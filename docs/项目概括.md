# Rag-Anything 多模态信息检索增强生成系统
## 2. 项目介绍
RagAnything 拟增强系统面对多模态信息时的检索增强生成能力。多模态信息包含图像、表格、公式等内容，并围绕 “解析 - 入库 - 检索- 生成” 构建端到端流水线。项目基于 LightRAG 实现，并采用 Mixins + Processors 的模块化设计，集成了批处理、多模态处理与查询等功能，且支持多种查询模式。

## 3. 核心功能

- 多模态文档解析与结构化：基于不同解析器（如 MinerU/Docling），设计支持多重文档的解析管线，统一输出结果为文本 + 多模态项。
- 多模态内容存储与知识图谱建模：基于 LightRAG ，对不同模态内容分发到对应 Processor 生成对应描述性文本，之后进行 chunk 生成、实体抽取与图谱合并。
- 查询侧设计分别基于 LLM 与 基于 VLM 的两种多模态查询：基于 LLM 的方法将多模态内容生成文本描述后由 LightRAG（LLM）检索与生成；基于 VLM 的方法利用 LightRAG 产出检索内容，提取其中图片并转为 base64，再交由 VLM 问答。


## 4. 性能优化
- 分层并发与限流：用 Semaphore 做文件级并发控制、文档内对多模态 items 并发处理（create_task + gather），提升吞吐并避免资源争抢。
- 去重与增量：基于 doc_status 与缓存命中跳过重复解析/入库，降低重复开销。
- 缓存加速：为多模态查询构造稳定 cache key，并统一管理 parse/LLM/多模态查询缓存，减少重复推理。
- 可恢复批处理：提供文件夹/列表批处理入口，记录成功/失败与错误信息，支持增量重跑与快速定位问题。
